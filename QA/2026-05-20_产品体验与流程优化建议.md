# 泥人剧场 - 产品体验与流程优化建议

> 日期：2026-05-20
> 分析方式：代码审查 + 前端页面分析（后端服务未启动，无法完整测试）

---

## 一、AI 配置中心问题与建议

### 1.1 配置引导缺失

**问题**：
- 新用户进入"AI 配置中心"时，面对4个标签页（文本/图像/视频/TTS）不知从何入手
- 没有说明配置顺序和依赖关系
- 缺少"快速开始"或"推荐配置"入口

**建议**：
```
添加"快速配置向导"：
1. 引导用户先配置文本大模型（用于剧本生成）
2. 然后配置图像生成（用于分镜图）
3. 最后可选配置TTS和视频

每个步骤显示推荐服务商和预估成本
```

### 1.2 ComfyUI 配置体验

**问题**：
- ComfyUI 默认 Base URL 是 `http://localhost:8188`，但没有连接测试功能
- 用户不知道 ComfyUI 是否正确安装和运行
- 工作流 JSON 配置对非技术用户不友好

**建议**：
```javascript
// 添加连接测试按钮
async function testComfyUiConnection() {
  try {
    const res = await fetch(`${form.value.baseUrl}/system_stats`)
    if (res.ok) {
      ElMessage.success('ComfyUI 连接成功！')
    }
  } catch {
    ElMessage.error('无法连接到 ComfyUI，请检查：\n1. ComfyUI 是否启动\n2. 端口是否正确\n3. 是否有防火墙限制')
  }
}
```

### 1.3 配置验证不足

**问题**：
- 保存配置时没有验证 API Key 是否有效
- 没有测试生成功能（图像配置有，但文本/视频/TTS没有）
- 错误提示不够具体

**建议**：
- 为每种配置类型添加"测试连接"按钮
- 保存前验证必填字段格式
- 提供更详细的错误信息和解决建议

### 1.4 配置状态不明确

**问题**：
- 配置列表只显示"默认"标签，没有显示配置状态（有效/无效/未测试）
- 用户不知道哪些配置可以正常使用

**建议**：
```
添加配置状态标签：
- ✅ 已验证（最近测试成功）
- ⚠️ 未测试（从未测试或超过30天）
- ❌ 失败（最近测试失败）
```

## 二、主流程贯通问题

### 2.1 流程入口不清晰

**问题**：
- 主流程：创意输入 → 剧本生成 → 分镜拆解 → 图片生成 → 视频生成 → 配音 → 合成导出
- 但页面导航没有明确体现这个流程
- 用户容易迷失在各个功能模块中

**建议**：
```
添加"项目进度看板"：
┌─────────────────────────────────────────────────────────────┐
│  创意输入 → 剧本生成 → 分镜拆解 → 图片生成 → 视频/配音 → 合成导出  │
│     ✅        ✅        ⏳        ○         ○          ○     │
└─────────────────────────────────────────────────────────────┘

点击每个步骤可以跳转到对应页面
显示当前步骤的完成度和下一步操作
```

### 2.2 沉浸式创作页面问题

**问题**（ImmersiveCreateView.vue）：
- 顶部有"开通会员"和"通知"按钮，但点击后是空操作（`noopVip`, `noopBell`）
- 剧集选择器在左侧边栏，不够直观
- 大纲确认后没有明确的下一步引导

**建议**：
```vue
<!-- 移除或实现空操作按钮 -->
<!-- 如果是测试版，显示"即将上线"提示 -->

<!-- 大纲确认后添加引导 -->
<div v-if="outlineConfirmed" class="next-step-guide">
  <h3>大纲已确认！</h3>
  <p>下一步：AI 将为您生成详细剧本</p>
  <el-button type="primary" @click="generateScript">
    生成剧本 →
  </el-button>
</div>
```

### 2.3 分镜制作页面问题

**问题**（StoryboardView.vue）：
- "动态镜头推荐"功能对新用户不够友好
- 推荐参数（最低分、最多应用）没有说明含义
- 缺少批量操作的确认提示

**建议**：
```
改进动态镜头推荐：
1. 添加推荐说明："系统根据镜头内容智能评估，分数越高越适合动态展示"
2. 默认参数预填合理值（最低分55，最多20个）
3. 批量操作前显示确认对话框，说明将影响多少个镜头
```

### 2.4 合成导出页面问题

**问题**（SynthesisView.vue）：
- 4个生成步骤（图片→动态→配音→合成）没有明确的依赖关系说明
- 用户可能在图片未生成时就尝试合成
- 缺少整体进度估算

**建议**：
```
改进生成流程：
1. 步骤间添加依赖检查（合成按钮在图片未就绪时禁用）
2. 显示预估时间："预计需要 5-10 分钟"
3. 添加"一键全部生成"按钮，按顺序自动执行
4. 生成过程中显示实时进度和当前步骤
```

## 三、用户体验问题

### 3.1 错误处理不友好

**现状**（request.ts）：
```typescript
// 502错误直接显示技术信息
ElMessage.error('Request failed with status code 502')
```

**建议**：
```typescript
const errorMessages: Record<number, string> = {
  401: '登录已过期，请重新登录',
  403: '没有权限执行此操作',
  404: '请求的资源不存在',
  500: '服务器内部错误，请稍后重试',
  502: '服务暂时不可用，请检查后端服务是否启动',
  503: '服务维护中，请稍后重试',
}

function getFriendlyErrorMessage(status: number, url: string): string {
  if (status === 502 && url.includes('/api/')) {
    return '后端服务未启动，请检查：\n1. MySQL 数据库是否运行\n2. 后端服务是否启动\n3. 端口 8080 是否被占用'
  }
  return errorMessages[status] || `请求失败 (${status})`
}
```

### 3.2 加载状态不明确

**问题**：
- 很多操作没有loading状态
- 长时间操作没有进度提示
- 用户不知道操作是否正在进行

**建议**：
- 所有API调用添加loading状态
- 长时间操作（>3秒）显示进度条
- 添加操作队列和取消功能

### 3.3 响应式设计问题

**问题**：
- 部分页面在小屏幕上布局混乱
- 表格没有横向滚动
- 弹窗在移动端显示不全

**建议**：
```css
/* 添加响应式断点 */
@media (max-width: 768px) {
  .config-card-body {
    grid-template-columns: 1fr;
  }

  .el-dialog {
    width: 95% !important;
    margin: 10px auto;
  }
}
```

## 四、功能完整性建议

### 4.1 缺少项目模板

**问题**：
- 每次创建项目都从零开始
- 没有预设的剧本模板或风格模板

**建议**：
```
添加项目模板库：
- 都市言情模板
- 悬疑惊悚模板
- 古装历史模板
- 玄幻奇幻模板

每个模板包含：
- 预设的风格描述
- 推荐的AI配置
- 示例剧本结构
```

### 4.2 缺少协作功能

**问题**：
- 项目只能单人操作
- 没有评论或批注功能
- 无法分享项目给他人

**建议**（中长期）：
- 添加项目分享链接
- 收藏和点赞功能
- 评论和反馈系统
- 团队协作空间

### 4.3 缺少素材库

**问题**：
- 每次生成图片/视频都是独立的
- 没有复用已生成素材的功能
- 角色图片无法跨项目复用

**建议**：
```
添加素材管理中心：
- 角色图片库（可复用）
- 场景背景库
- 音效/音乐库
- 常用提示词库
```

### 4.4 缺少版本管理

**问题**：
- 剧本修改后无法回退
- 分镜调整没有历史记录
- 无法比较不同版本的差异

**建议**：
- 添加版本历史功能
- 支持版本对比和回退
- 自动保存和手动保存

## 五、性能优化建议

### 5.1 前端性能

**问题**：
- 大量组件一次性加载
- 图片没有懒加载
- 没有虚拟滚动

**建议**：
```typescript
// 路由懒加载
const StoryboardView = () => import('./views/storyboard/StoryboardView.vue')

// 图片懒加载
<img v-lazy="shot.imageUrl" />

// 虚拟滚动（大量分镜时）
<virtual-list :data="storyboards" :item-height="200" />
```

### 5.2 API 请求优化

**问题**：
- 没有请求缓存
- 没有请求防抖
- 重复请求没有取消

**建议**：
```typescript
// 请求缓存
const cache = new Map()

async function cachedRequest(url: string, ttl = 60000) {
  const cached = cache.get(url)
  if (cached && Date.now() - cached.time < ttl) {
    return cached.data
  }
  const data = await request.get(url)
  cache.set(url, { data, time: Date.now() })
  return data
}

// 防抖
import { useDebounceFn } from '@vueuse/core'
const debouncedSearch = useDebounceFn(searchProjects, 300)
```

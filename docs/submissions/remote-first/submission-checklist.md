# Remote-First CASR Checklist

## Remove On-Site Dependencies

- [ ] Do not target demo/poster tracks that require an in-person booth.
- [ ] Do not target proceedings where non-attendance can remove the paper.
- [ ] Do not rely on a live conference presentation for credibility.
- [ ] Use an online video and repository as the primary demonstration medium.

## arXiv Preparation

- [ ] Convert `paper/casr-arxiv-paper.md` into a full English LaTeX manuscript.
- [ ] Use a standard arXiv-compatible LaTeX setup.
- [ ] Add figures for architecture, shot graph, UI, strategy tree, and case-study comparison.
- [ ] Include `.bib` or generated `.bbl` files in the source package.
- [ ] Preview the generated PDF before final submission.
- [ ] Choose the final category after checking the current arXiv category guidance.

## Research Release

- [ ] Record the remote demo video.
- [ ] Add the video link to README.
- [ ] Add a paper link once the preprint is public.
- [ ] Add screenshots or architecture diagrams to `docs/`.
- [ ] Prepare a one-minute technical review version of the demo.

## Research Positioning

- [ ] Research summary: "Designed and implemented CASR, a continuity-aware self-repair algorithm for AI short-drama production pipelines, covering failure attribution, cost-aware policy search, and explainable repair planning."
- [ ] Research artifact headline: "Full-stack AI video production system with research-grade self-repair algorithm."
- [ ] Review pitch: "I built the production system and the algorithm layer that diagnoses and repairs generation failures instead of blindly rerunning tasks."

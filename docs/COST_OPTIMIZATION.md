## 二、当前系统生成链路

```
用户输入创意
    ↓
[1] 剧本生成 (TextAI: GPT-4o / 豆包 / 通义千问)
    ↓
[2] 分镜拆解 (TextAI: 剧本 → JSON 分镜列表)
    ↓
[3] 分镜图片生成 (ImageAI: DALL-E 3 / 通义万相)
    ├── 每个镜头生成 1 张 1024×1792 图片
    └── 风格: vivid, 垂直构图 9:16
    ↓
[4] 配音生成 (TTS: OpenAI / 火山引擎 / MiniMax)
    ├── 每个镜头生成 1 段语音 (MP3)
    └── 文本 = 旁白 + 台词
    ↓
[5] 视频合成 (FFmpeg, 本地处理, 免费)
    ├── 图片 + 音频 → Ken Burns 缩放特效
    ├── 输出: 1080×1920, 25fps, H.264+AAC
    └── 拼接所有镜头 → 完整单集 MP4
```

## 五、成本占比分析

### 当前方案（FFmpeg 合成，无 AI 视频）

```
文本生成:    0.06%  ← 几乎可忽略
图片生成:   99.55%  ← 绝对主力成本
TTS配音:     0.39%  ← 极低
视频合成:    0.00%  ← FFmpeg 本地免费
```

### 如加入 AI 视频生成

```
视频生成:   94.89%  ← 成本爆炸
图片生成:    5.06%
文本生成:    0.00%
TTS配音:     0.02%
```

**关键发现**：
1. **图片生成**是当前方案的核心成本（占99.5%+）
2. **AI 视频生成**极其昂贵（单集 ¥146-¥288），50 集高达 ¥7,344-¥14,400
3. 文本生成和 TTS 成本几乎可以忽略

## 六、大幅缩减成本的优化策略

### 策略 1：场景图片复用（预计节省 40-60% 图片成本）

**原理**：短剧中同一场景、同一角色会反复出现。相同场景 + 相同角色 + 相似构图的镜头可以复用同一张图片。

**实现方式**：
- 按 `sceneId + characterId + cameraAngle` 建立缓存键
- 相同缓存键的镜头复用已生成的图片
- 只对首次出现的唯一组合生成新图

**效果预估**：
- 原始：4,800 张图片
- 优化后：~2,000 张（减少 58%）
- 节省：¥768 → ¥320 = 节省 **¥448**

### 策略 2：智能分辨率分级（预计节省 20-30%）

**原理**：不是所有镜头都需要最高画质。
- **远景/全景**（wide/overhead）：使用 720P 即可
- **中景**（medium）：使用标准 1024×1024
- **特写**（close-up/pov）：使用高清 1024×1792

**效果预估**：
- 特写(30%): 1024×1792 → $0.12
- 中景(40%): 1024×1024 → $0.08
- 远景(30%): 512×912 → $0.04 (降低质量换成本)
- 平均单价从 $0.12 降至 ~$0.08
- 节省约 **33%**

### 策略 3：混合视频策略（AI 视频仅用于关键镜头）

**原理**：不是所有镜头都需要 AI 视频生成。

| 镜头类型 | 占比 | 视频方式 | 成本 |
|----------|------|----------|------|
| 对话/旁白（静态）| 50% | FFmpeg + Ken Burns | ¥0 |
| 缓慢运动 | 20% | FFmpeg + 高级特效 | ¥0 |
| 动作/转场（动态）| 30% | AI 视频生成 | 可灵/Seedance |

**效果预估**（以可灵为例）：
- 原始：480s × ¥0.6 = ¥288/集
- 优化后：仅 30% 使用 AI → 144s × ¥0.6 = ¥86.4/集
- 50 集：¥14,400 → ¥4,320 = 节省 **¥10,080**

### 策略 4：使用国产低成本 API 替代

| 替换 | 原始成本 | 优化后 | 节省率 |
|------|----------|--------|--------|
| GPT-4o → 通义千问-turbo | ¥31.50 | ¥0.50 | 98.4% |
| DALL-E 3 → 通义万相 | ¥4,176 | ¥768 | 81.6% |
| OpenAI TTS → 火山引擎 | ¥31.50 | ¥3.00 | 90.5% |

### 策略 5：跨集素材复用

**原理**：50 集短剧中，主要角色和场景是固定的。

- **角色立绘**：每个角色只生成一次基础形象
- **场景背景**：每个场景只生成一次背景图
- **跨集复用**：相同角色 + 场景组合可直接复用之前集的图片

**效果预估**：
- 假设 50 集中有 10 个主要场景 × 5 个主要角色 = 50 种基础组合
- 每种组合 × 5 个角度 = 250 张基础图
- 复用率可达 **80%+**

### 策略 6：预览模式 & 正式模式分离

- **预览模式**：使用低分辨率(512×512) + 便宜模型，快速预览效果
- **正式模式**：确认无误后再用高质量模型生成
- 避免因调整剧本反复生成高清图片的浪费
